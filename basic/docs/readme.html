D2K Basic 4.1.2

This is D2K Basic 4.1.2. This version sees a number of changes from the previous 4.1.1 release. Java 1.5.0 or greater is now required.

D2K Infrastructure:
  • Experimental support for SSL sockets with D2K server communications
  • Enhancements to ncsa.d2k.batch.BatchInterface
    • Documentation
    • New methods to make it easier to use
  • Libraries that modules depend on can now be located in <d2k home>/lib/ext, instead of in the modules directory. Moving libraries out of the modules directory speeds up load time
  • Enhancements to class loader protocol
    • Remote resource loading
  • Added removeResource() method to RootModule to remove an object from the ResourceManager
  • Numerous documentation updates
  • HeadlessUIModule: changed supressGui property to suppressGui
  • machines.txt (for distributed computing) is now in the D2K install directory rather than the user home directory
  • Enhancements to nested itineraries:
    • Fixed ordering of nested itinerary inputs
    • Brushing the ports of a nested itinerary will show the name of the port and the name of the module inside that the port comes from
    • Fixed bug in which certain nested modules could execute more than once
    • Now possible to nest itineraries within other nested itineraries
D2K Toolkit:
  • Expanded use of anti-aliasing for text
D2K Basic:
  • Moved Column, NumericColumn, TextualColumn interfaces from ncsa.d2k.modules.core.datatype.table.basic package to ncsa.d2k.modules.core.datatype.table
  • Added getTableFactory() to Table interface
  • New sparse table implementation in ncsa.d2k.modules.core.datatype.table.sparse2 package.
  • Fix to ChooseAttributes: when attribute names were saved so the module could run headless, they were all capitalized. This could cause problems, so the capitalization was removed
  • Updated the included JUNG and Prefuse libraries.
  • Added a number of modules related to Prefuse-based graph visualization.
  • Included a Java 3D rule visualization.
  • The module ncsa.d2k.modules.core.control.Sink was deprecated and replaced with ncsa.d2k.modules.core.control.DisposeObject
  • ncsa.d2k.modules.core.optimize package was refactored:
    The following modules were deprecated and replaced:
    • MultiTrainTestBiasEvaluator is replaced with ncsa.d2k.modules.core.optimize.evaluators.ModelingParametersEvaluator
    • RandomGriddedSampling is replaced with ncsa.d2k.modules.core.optimize.samplers.GriddedSampler
    • ncsa.d2k.modules.core.optimize.random.RandomSampling is replaced with ncsa.d2k.modules.core.optimize.samplers.RandomSampler.java
    • ncsa.d2k.modules.core.optimize.random.UniformSampling is replaced with ncsa.d2k.modules.core.optimize.optimizer.RandomOptimizer.java
    The following modules were moved to different packages:
    • CreateExample was moved to ncsa.d2k.modules.core.optimize.examples
    • SimpleModelEvaluator was moved to ncsa.d2k.modules.core.optimize.evaluators
    • ncsa.d2k.modules.core.optimize.random.ExampleCollector was moved to ncsa.d2k.modules.core.optimize.examples
    • ncsa.d2k.modules.core.optimize.random.ExampleCollector was moved to ncsa.d2k.modules.core.optimize.examples
T2K:
  • Internal structural changes to ncsa.d2k.modules.t2k.corpora.Document, Annotation, AnnotationSet
  • Set serialVersionUID of all Documents
  • GateDocumentViewer, GateDocumentsViewer: clear doc list at end of execution
  • Updated Gate modules to work with distributed computing
  • Added reentrant versions of Gate modules

Changes made in 4.1.1

D2K Basic 4.1.1 saw several new features introduced since the previous 4.0.2 release. There has been significant work improving batch mode so that this version can run on the batch systems. There have also been significant overall improvements in the way nested itineraries are used and edited.

The dialog to update modules, data and itineraries no longer exits. Now the modules, data files and itineraries are always installed if an update is available.

The D2K properties file in no longer read from the users home directory at all. It is always read from the directory where the application exists.

There is a new Auto Checkpointing feature that will allow a user to set up an automatic checkpoint. The user will specify a number of minutes between checkpoints, and as nearly as possible, the itinerary execution will pause, checkpoint and continue. This occurs automatically.

Nested itinerarys have changed significantly. Now, nested itineraries can behave like ReentrantComputeModules. They can clone themselves and execute in parallel on different parcels of data. There is a property of the nested itinerary module that will enable this feature. The user specifies the number of copies of the itinerary to create, and the system does the rest.

Also, nested itinerary editing has changed. Now nested itineraries appear in the same window as the outter itinerary. Many bugs in working with deeply nested itineraries has also been fixed.

Logging is now fully supported in D2K. If you change the logging level in the preferences, the changes take effect immediately, there is no need to restart D2K.

There is an editor included for the "machines.txt" file. Recall, this file defines what machines are available for distributed computing if the Jini URL is set to test.

There is a new feature (as of B3) where a module enables as the result of some external event. In order for this to work, the appropriate logic must be implemented in the isReady method of the module. Also, the module must implement the AlwaysReady interface. Beyond this, simply develop code that, when the external event occures, call the checkMe method of the modules execution manager. This functionality is only available for ExecModules.

There are now two jar files instead of one. The infrastructure.jar file contains the d2k execution engine and infrastructure. The toolkit.jar contains the D2K Toolkit code. There are also now two entry points the differing D2K applications distributed. To run D2K in batch mode, run the class "ncsa.d2k.batch.CommandLineD2K". To run the Toolkit, run "ncsa.d2k.gui.ToolKit".
